Quick Estimate Module Instruction

Module name: Quick Estimate
Enabled: true
Product scope: Business Autopilot only

Required fields:
- mobile
- client_name
- at least one item
- amount

Optional fields:
- email
- address
- gst_number
- notes
- assigned_to_user
- payment_status
- delivery_status

Quick Estimate rules:
- Estimate number format is QE-0001, QE-0002.
- Estimate number is generated by backend, not AI.
- Mobile number and client name are mandatory for create.
- Email, address, GST number are optional.
- If mobile already exists in same organization, backend will use existing client.
- If mobile is new, backend will create new client using mobile and client name.
- Same mobile should not create duplicate client.
- If user mentions QE-0001, treat it as existing estimate reference.
- For editing, adding items, status update, or assignment, estimate_number is mandatory.
- If estimate number is missing for edit command, ask: "Yentha estimate number edit pannanum? Example QE-0001."

Payment status allowed values:
- unpaid
- advance_paid
- partial_paid
- paid
- cancelled
- refunded

Delivery status allowed values:
- pending
- design_pending
- design_completed
- printing
- ready
- delivered
- cancelled

Supported intents:
- quick_estimate_create
- quick_estimate_edit
- quick_estimate_add_item
- quick_estimate_remove_item
- quick_estimate_update_payment_status
- quick_estimate_update_delivery_status
- quick_estimate_assign_user
- quick_estimate_whatsapp_share
- ask_missing_field
- unknown

Conversation memory / slot filling:
- User can give details all at once or one by one.
- Maintain pending state per organization and user.
- Pending state fields: active_module, active_intent, collected_data, current_step, last_estimate_number, last_estimate_id.
